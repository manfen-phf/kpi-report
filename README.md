# 商品运营分析报告（自动生成 · 每日更新）

每天自动读取最新业务数据，生成「分区进度 / 玉林商进度」双板块可视化报告，并部署到 GitHub Pages 在线查看。

## 数据来源（自动读取最新文件）
- **结果指标（每日）**：`E:\0总商维度数据\KPI绩效\结果指标（每日）` —— 取数以表内 **G4 列日期**为准（文件名"X月Y日更"仅为更新日，非数据日）。
- **过程指标（看板）**：`E:\0总商维度数据\KPI绩效\过程指标（看板）` —— 重点货盘攻克率（H 列）。
- **覆盖率日维度**：`E:\0总商维度数据\KPI绩效\001重点货盘日报输出\商家参与优惠活动覆盖率\日维度数据`。

## 日常更新（一键）
1. 把新一天的 Excel 放入上述对应目录（保持原命名规律）。
2. 双击 **`update.bat`**（或在本目录运行 `python generate_report.py`）。
3. 脚本会重新生成 `index.html`（最新）与 `商品运营分析报告-MMdd.html`（按日存档），并自动提交推送到 GitHub Pages。

## 计分规则
- 综合得分 = 拼好饭 50% + 货盘 40% + 月周均覆盖率 10%（P 城市东兰/凤山 = 拼好饭 50% + 货盘 50%，不考核覆盖率）。
- 时间进度 = 数据日 / 当月天数（7 月 31 天）。
- 风险：⚠️ 警告（预估总得分 < 90）｜ 💰 罚款（过程指标预估 < 45 且不可减免）｜ 🛡️ 可减免（拼好饭预估完成率 ≥ 105%）。
